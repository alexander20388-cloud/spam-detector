🧮 Cara Kerja Teorema Bayes untuk Deteksi Spam

1. Apa itu Teorema Bayes?

Teorema Bayes adalah fondasi matematika untuk memperbarui keyakinan (probabilitas) berdasarkan bukti baru. Diformulasikan oleh Thomas Bayes (1763), teorema ini menyatakan:

P(A | B) = P(B | A) × P(A) / P(B)

Di mana setiap komponen memiliki makna spesifik dalam konteks deteksi spam:

NotasiNamaArti dalam Deteksi Spam
P(A|B)PosteriorProbabilitas pesan adalah spam, setelah melihat isi pesannya
P(B|A)LikelihoodSeberapa sering kata-kata ini muncul di pesan spam
P(A)PriorProporsi spam di dataset — sebelum melihat isi pesan
P(B)EvidenceProbabilitas kemunculan kata-kata tersebut secara umum

2. Penerapan ke Teks: Naive Bayes Classifier

Untuk pesan berisi kata-kata w₁, w₂, ..., wₙ, kita ingin menghitung:

P(Spam | w₁, w₂, ..., wₙ) ∝ P(Spam) × P(w₁|Spam) × P(w₂|Spam) × ... × P(wₙ|Spam)

Kata "Naive" merujuk pada asumsi bahwa setiap kata bersifat independen satu sama lain. Meskipun asumsi ini tidak 100% realistis (kata "hadiah" dan "klaim" sering muncul bersama), pendekatan ini terbukti sangat efektif dan efisien dalam praktik.

Untuk menghindari underflow numerik (perkalian banyak angka kecil), perhitungan dilakukan dalam skala logaritmik:

log P(Spam|pesan) = log P(Spam) + Σ log P(wᵢ | Spam)

3. Proses Lengkap — 5 Langkah

  1. Preprocessing — teks dinormalisasi: huruf kecil semua, URL diganti token url, angka diganti angka, tanda baca dihapus.

  2. TF-IDF Vectorization — setiap kata dikonversi menjadi bobot numerik. TF-IDF memberi nilai tinggi untuk kata yang sering muncul di dokumen ini tapi jarang di seluruh dataset — mengurangi pengaruh kata umum seperti "dan", "yang", "the".

  3. Hitung Prior P(Kelas) — berapa proporsi spam dan ham di data training. Jika 50% spam: P(Spam) = 0.5, P(Ham) = 0.5.

  4. Hitung Likelihood P(Kata|Kelas) — untuk setiap kata di pesan, hitung seberapa sering kata tersebut muncul di pesan spam vs ham. Gunakan Laplace smoothing (α=0.5) agar kata baru tidak menghasilkan probabilitas nol.

  5. Hitung Posterior & Normalisasi — gabungkan prior dan likelihood, bandingkan P(Spam|pesan) vs P(Ham|pesan). Kelas dengan nilai tertinggi adalah prediksi. Normalisasi menghasilkan confidence score (0–100%).

4. Contoh Perhitungan Manual

Misalkan pesan: "Klik sekarang menang hadiah"

Prior: P(Spam) = 40/80 = 0.500 P(Ham) = 40/80 = 0.500 Likelihood (log): log P("klik" | Spam) = -3.21 vs log P("klik" | Ham) = -5.89 log P("menang" | Spam) = -3.45 vs log P("menang" | Ham) = -6.12 log P("hadiah" | Spam) = -2.98 vs log P("hadiah" | Ham) = -6.45 Log-posterior (tidak ternormalisasi): Spam: log(0.5) + (-3.21) + (-3.45) + (-2.98) = -10.33 Ham: log(0.5) + (-5.89) + (-6.12) + (-6.45) = -19.15 Normalisasi → P(Spam|pesan) ≈ 0.9998 → SPAM ✓

ℹ️ Tentang SpamShield

📐 Teorema Bayes

Klasifikasi didasarkan sepenuhnya pada Teorema Bayes: P(Spam|Pesan) = P(Pesan|Spam) × P(Spam) / P(Pesan). Model menghitung probabilitas posterior untuk setiap kelas.

📝 Ekstraksi Fitur

Teks dikonversi ke vektor numerik menggunakan TF-IDF dengan bigram, memberi bobot lebih pada kata unik yang membedakan spam dari ham.

🌐 Bilingual

Dataset mencakup Bahasa Indonesia dan Inggris — masing-masing 40 spam + 40 ham, total 80 pesan pelatihan.

🔄 Preprocessing

Lowercasing → normalisasi URL → normalisasi angka → hapus tanda baca → tokenisasi TF-IDF.

📊 Evaluasi

Accuracy, Precision, Recall, F1-Score, dan Confusion Matrix. Pembagian data 80% training / 20% testing dengan stratified split.

⚙️ Tech Stack

Python · Flask · scikit-learn · pandas · matplotlib · seaborn
